## Flagship — VIRA

**Active · live public deployment**

VIRA turns authoritative football observations into synchronized multiplayer challenges. The browser never decides the deadline, result or ranking: the server admits private answers, locks each round, resolves against eligible TxLINE evidence and rebuilds public projections from an append-only, hash-chained ledger.

[Live app](https://vira.snelabs.space/?lang=en) · [Source](https://github.com/4LFR3Dv1/VIRA-) · [Demo V2](https://www.youtube.com/watch?v=LnOd2kWTiGA) · [Evaluation guide](https://vira.snelabs.space/help?lang=en) · [Public playback evidence](https://vira.snelabs.space/public/playback)

Verified public surfaces include a passing repository verification gate, live health/readiness endpoints, deterministic replay tests, a two-device browser flow and an optional asynchronous Solana devnet commitment. The permanent judge path uses a disclosed, sanitized captured TxLINE fixture; the current event store is deliberately single-writer.

`TypeScript` · `React` · `Node.js` · `SSE` · `Event Sourcing` · `Playwright` · `Solana` · `Docker`
